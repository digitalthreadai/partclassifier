Strategy & Features

Architecture, feature map, tech stack, and development roadmap. 7,031 LOC across 17 modules.

PartClassifier is a production-grade AI agent for mechanical part classification and attribute extraction, built for industrial engineering teams managing large inventories of fasteners, bearings, fittings, sensors, and other mechanical components. It automates the process of looking up part specifications from distributor websites, extracting structured attributes, and preparing classification-ready Excel outputs for Teamcenter PLM import.

The agent combines deterministic pattern matching (95% accuracy), regex pre-extraction, and LLM intelligence with multi-source data retrieval -- distributor APIs (DigiKey OAuth2, Mouser API, McMaster mTLS), stealth web scraping (CloakBrowser with 33 C++ patches), and intelligent caching -- to process thousands of parts with minimal human intervention. An Excel-based schema (ClassificationSchema.xlsx) with 81 classes, 46 attributes, and 169 aliases drives the entire pipeline.

Core Pipeline Features

7-Tier Web Search

web_scraper.py, api_sources.py, stealth_scraper.py

DigiKey (OAuth2) / Mouser (API key) / McMaster (mTLS) → CloakBrowser stealth → URL cache (30-day TTL + bad-entry eviction) → DuckDuckGo + curl_cffi (Chrome 124 TLS) → stealth fallback. Preferred trusted domains scored higher.

Content Cleaner

content_cleaner.py

HTML table extraction as structured key-value data. Handles header-row and two-column patterns. Smart truncation: 3K tables / 5K text / 8K combined. Specs prioritized over navigation boilerplate.

Deterministic Classification

class_extractor.py

Pattern-based classification from web content with 95% accuracy. Scans breadcrumbs, category labels, titles, URL paths. 90+ class aliases. Parent-child specificity resolution. No LLM needed when confident (score ≥ 4).

LLM Part Classification

part_classifier.py

81 categories. Batch classification: 100 parts per CLI call (75x faster), 50 per API call. LLM cache with 90-day TTL. Individual fallback for batch parse failures.

Regex Pre-Extraction

regex_extractor.py

Pattern-based extraction from tables, key-value pairs, standards (DIN/ASME/ISO), materials (18-8 SS, 304L, Viton). Builds label patterns from ALIASES dictionary. Agreement tracking vs LLM results.

LLM Hybrid Extraction

attribute_extractor.py

Hybrid prompt: schema attrs as priority hints + maximize coverage. LLM validates regex pre-extracted values, fills gaps. Unit conversion (inches/mm). Validation retry for >50% missing attrs. Temperature=0.

Excel-Based Schema

attr_schema.py

ClassificationSchema.xlsx: 81 classes, 46 canonical attributes, 169 aliases. Fallback to hardcoded schemas. 500+ alias mappings including DigiKey/Mouser API parameter names. Fuzzy schema matching.

Per-Class Excel Output

excel_handler.py

Groups parts by type. One .xlsx per class with TC Class ID column for Teamcenter integration. Consistent canonical column headers, schema-ordered. Styled headers (blue fill, white bold).

Execution Modes

API Key CLI (main.py)
  • Sequential processing with asyncio
  • 8 LLM providers via .env
  • Resume via progress.json checkpoint after each part
  • Excel writes every 10 parts
  • Retry with exponential backoff for timeouts and rate limits
  • CLI flags: --input, --output, --no-cache, --clear-cache
  • Best for: small batches, testing, debugging
Claude Code CLI (main_cc.py)
  • Zero API key requirement (uses claude CLI on PATH)
  • Batch classification: 100 parts per CLI call (75x faster)
  • Parallel workers (default 4, --workers N)
  • Resume via progress_cc.json (or provider-specific progress files)
  • Per-part error isolation (one failure doesn't stop the batch)
  • Intermediate Excel writes every 50 parts
  • Ctrl+C safe (saves progress + partial output)
  • Best for: large batches (1K-20K+ parts), enterprise environments with Claude Code on Azure
Streamlit Web UI (app.py)
  • Interactive provider/model configuration in sidebar
  • File picker with data preview
  • Live progress bar and per-part results
  • Download buttons for output files
  • Auto-writes .env from sidebar configuration
  • Best for: non-technical users, demos

Infrastructure Features

LLM Response Cache

llm_cache.py

Thread-safe cache with atomic writes (Windows-safe via tempfile + os.replace). Classification: 90-day TTL (MD5 key). Extraction: 30-day TTL (mfg_part_num + class + content hash). Eliminates redundant LLM calls.

URL Caching

shared.py

Shared url_cache.json across modes. 30-day TTL with bad-entry eviction. Coverage improves with each run as cache warms. Atomic writes for Windows safety. Safe to commit.

Metrics Tracking

metrics.py

Quality metrics, cache hits, regex/LLM agreement rate, LLM call counts, timing (seconds/part). Appends to metrics_history.json for trend analysis across runs.

CloakBrowser Stealth

stealth_scraper.py

33 C++ source-level patches to Chromium. Bypasses Cloudflare, reCAPTCHA v3 (0.9 score), FingerprintJS, Turnstile. Fresh context per scrape. Direct URL patterns for McMaster/Fastenal/Grainger. Optional.

curl_cffi TLS Bypass

web_scraper.py

Chrome 124 TLS fingerprint impersonation (JA3/JA4). Browser-like headers (Accept-Language, Accept). Rate limiting: 1.5s DuckDuckGo, 30 req/min Mouser.

Manufacturer Rotation

shared.py

Round-robin interleave by manufacturer. Largest buckets spread widest. Spaces out requests to same site to reduce bot detection risk.

Production Hardening

  • Windows UTF-8 encoding fix: prevents encoding errors in Excel output and console
  • Resume capability: checkpoint saved after each part (progress.json / progress_cc.json), deleted on completion
  • Validation retry: re-extraction for missing attrs when >50% of schema attrs unfilled
  • Fallback to part name: mines dimensions from part name when no web content found
  • Per-part error isolation: one failed part doesn't stop the batch
  • Atomic writes: Windows-safe via tempfile + os.replace for all JSON caches
  • 21 production-ready bug fixes: edge case handling for malformed HTML, encoding issues, API timeouts, graceful degradation when optional dependencies missing
