Strategy & Features

Architecture, feature map, tech stack, and development roadmap.

PartClassifier is an AI-powered mechanical parts classification agent designed for industrial engineering teams managing large inventories of fasteners, bearings, fittings, sensors, and other mechanical components. It automates the process of looking up part specifications from distributor websites, extracting structured attributes, and preparing classification-ready Excel outputs for Teamcenter PLM import.

The agent combines deterministic pattern matching, regex pre-extraction, and LLM intelligence with multi-source data retrieval -- distributor APIs, stealth web scraping, and intelligent caching -- to process thousands of parts with minimal human intervention.

Core Pipeline Features

Multi-Tier Web Search

web_scraper.py, api_sources.py, stealth_scraper.py

DigiKey/Mouser/McMaster APIs → CloakBrowser → URL cache → DuckDuckGo + curl_cffi → stealth fallback. Preferred trusted domains scored higher.

Deterministic Classification

class_extractor.py

Pattern-based classification from web content. Scans breadcrumbs, category labels, titles, URL paths. 90+ class aliases. No LLM needed when confident.

LLM Part Classification

part_classifier.py

60+ categories. Batch classification: 50 parts per LLM call (95% token savings). LLM cache with 90-day TTL.

Regex Pre-Extraction

regex_extractor.py

Pattern-based extraction from tables, key-value pairs, standards (DIN/ASME/ISO), materials. Agreement tracking vs LLM results.

Table-Aware Content Extraction

content_cleaner.py

Extracts HTML tables as structured key-value data. Handles header-row and two-column patterns. Tables prioritized over narrative text.

LLM Validation & Gap Fill

attribute_extractor.py

LLM validates regex pre-extracted values and fills gaps. Automatic unit conversion (inches/mm). Retry for missing attributes. Temperature=0.

Canonical Normalization

attr_schema.py

60+ part classes, 15 class-specific schemas, 500+ alias mappings including DigiKey/Mouser API parameter names. Fuzzy schema matching.

Per-Class Excel Output

excel_handler.py

Groups parts by type. One .xlsx per class with consistent canonical column headers. Styled headers (blue fill, white bold).

Execution Modes

API Key CLI (main.py)
  • Sequential processing with asyncio
  • 7 LLM providers via .env
  • Resume via progress.json checkpoint
  • Excel writes every 10 parts
  • Retry with exponential backoff
  • CLI flags: --input, --output, --no-cache, --clear-cache
Claude Code CLI (main_cc.py)
  • Zero API key requirement (uses claude CLI)
  • Batch classification: 100 parts per CLI call
  • Parallel workers (default 4, --workers N)
  • Resume via progress_cc.json
  • Per-part error isolation
  • Ctrl+C safe (saves progress + partial output)
  • Best for: large batches (1K-20K+ parts), enterprise
Streamlit Web UI (app.py)
  • Interactive provider/model configuration
  • File picker with data preview
  • Live progress bar and per-part results
  • Download buttons for output files
  • Auto-writes .env from sidebar

Infrastructure Features

LLM Response Cache

llm_cache.py

Thread-safe cache with atomic writes. Classification: 90-day TTL. Extraction: 30-day TTL. Keyed on normalized input MD5 hash.

URL Caching

shared.py

Shared url_cache.json across modes. 30-day TTL with timestamp migration. Atomic writes for Windows safety.

Metrics Tracking

metrics.py

Quality, cache hits, regex/LLM agreement rate, LLM call counts, timing. Appends to metrics_history.json for trends.

CloakBrowser Stealth

stealth_scraper.py

33 C++ patches to Chromium. Bypasses Cloudflare, reCAPTCHA v3, FingerprintJS. Fresh context per scrape. Optional.

curl_cffi TLS Bypass

web_scraper.py

Chrome 124 TLS fingerprint (JA3/JA4). Browser-like headers. Rate limiting for DuckDuckGo and Mouser.

Manufacturer Rotation

shared.py

Round-robin interleave by manufacturer. Largest buckets spread widest. Reduces bot detection risk.
