Part Classification Agent v2.0

Production-grade AI agent for mechanical part classification and attribute extraction. v2.0 -- rearchitected pipeline with attribute-fit validation. 7,031 LOC across 18 modules.

PartClassifier reads mechanical parts from an Excel file, searches distributor APIs and the web for specifications, classifies each part into 93 categories with 95% accuracy, extracts structured attributes via regex pre-extraction + LLM validation using hybrid prompts, and writes one Excel output file per part class with TC Class ID -- ready for Teamcenter PLM import.

API Key CLI

python main.py
9 LLM providers via .env configuration. --fresh flag for clean runs

Claude Code CLI

python main_cc.py
Zero API keys, batch 100/call, parallel workers

Streamlit Web UI

streamlit run app.py
Interactive UI for non-technical users

Pipeline (7 Steps): Read → Search → Classify → Blind Extract → Validate → Extract → Write


  

    
StepActionModule(s)

    
1. ReadRead input Excel, parse part numbers, manufacturer info, unit of measureexcel_handler.py

    
2. SearchMulti-tier search: 7 tiers from APIs to fallback. DigiKey (OAuth2), Mouser (API key), McMaster (mTLS), CloakBrowser, URL cache, DuckDuckGo + curl_cffi, stealth fallbackweb_scraper.py, api_sources.py, stealth_scraper.py

    
3. ClassifyDeterministic from web content (95% accuracy) → LLM cache → batch LLM (100/call)class_extractor.py, part_classifier.py

    
4. Blind ExtractRegex pre-extraction from HTML tables, key-value patterns, standards (DIN/ASME/ISO), materials -- before classification is knownregex_extractor.py, content_cleaner.py

    
5. ValidateAttribute-fit validation: check blind-extracted attributes against class schema, flag mismatches, reclassify if neededclass_validator.py

    
6. ExtractLLM validates pre-extracted values + fills gaps via hybrid prompt (priority schema hints + maximize coverage), temperature=0. Unit conversion with as-is mode supportattribute_extractor.py

    
7. WriteNormalize 500+ alias mappings → canonical names, schema-ordered per class. Write per-class Excel with TC Class ID, canonical columns, styled headersattr_schema.py, excel_handler.py

Adaptive Search


  
The agent finds sources dynamically based on part number presence in content -- no dependency on hardcoded domain lists.


  
    
    
  • Dynamic source discovery: DuckDuckGo results are scored by whether the manufacturer part number actually appears in the page content. The best page wins regardless of domain.
    • 
    
  • No hardcoded domains: Works for any manufacturer at scale (30K+ parts) without maintaining a list of trusted sites. New distributors and niche suppliers are discovered automatically.
    • 
    
  • Part number verification: Pages are ranked by a _spec_score() function that checks for the presence of the searched part number, specification tables, and technical content.
    • 
    
  • URL cache as learned knowledge: Once a good source is found, it is cached for 30 days. Subsequent runs skip search entirely for cached parts, and bad entries are evicted automatically.
    • 
  


  
This approach scales to any industry vertical -- fasteners, bearings, electronics, semiconductor equipment -- without per-domain configuration.


  
JSON Schema & PLMXML Import


  
The classification schema is now driven by Teamcenter-compatible JSON files:


  
    
    
  • schema/Classes.json -- Hierarchical class tree with 93 classes. ICM-format IDs matching Teamcenter classid. Parent-child tree with attribute inheritance. Class aliases for web content classification.
    • 
    
  • schema/Attributes.json -- Attribute dictionary with 46 attributes. 5-digit numeric IDs matching Teamcenter attribute IDs. Includes unitOfMeasure, range, LOV values, and keyLOVID.
    • 
    
  • schema/aliases.json -- Auto-generated alias mappings for attribute normalization.
    • 
    
  • schema/classification_hints.json -- LLM-generated classification hints keyed by class name, used to improve classification accuracy.
    • 
  


  
Attribute inheritance: Child classes automatically inherit all ancestor attributes. LOV normalization: Extracted values are matched to Teamcenter LOV entries (e.g., "Stainless Steel" → "StainlessSteel"). Schema files are the single source of truth -- no Excel or hardcoded fallbacks.


  
PLMXML Converter


  
The included plmxml_to_json.py converts Teamcenter PLMXML exports to the JSON schema format:


  
python plmxml_to_json.py --plmxml export.xml --sml attributes.xml --output schema/
python plmxml_to_json.py --plmxml export.xml --dry-run     # preview without writing
python plmxml_to_json.py --demo                             # generate sample output


  
Parses <AdminClass> tags, reconstructs hierarchical trees, and resolves attribute→format→KeyLOV chains. Zero external dependencies (stdlib only). CLI modes: --plmxml, --sml, --output, --dry-run, --merge, --demo.
