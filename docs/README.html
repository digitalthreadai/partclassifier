Part Classification Agent

AI-powered mechanical parts classification, web scraping, and structured attribute extraction.

PartClassifier reads a list of mechanical parts from an Excel file, searches distributor APIs and the web for specifications, classifies each part into 60+ categories, extracts structured attributes with regex pre-extraction and LLM validation, and writes one Excel output file per part class -- ready for Teamcenter PLM import.

API Key CLI

python main.py
7 LLM providers via .env configuration

Claude Code CLI

python main_cc.py
Zero API keys, batch + parallel workers

Streamlit Web UI

streamlit run app.py
Interactive UI for non-technical users

Quick Start

Option A: Claude Code CLI (no API keys)

git clone https://github.com/digitalthreadai/partclassifier.git
cd partclassifier
pip install -r requirements.txt
python main_cc.py

Option B: API Key Mode

git clone https://github.com/digitalthreadai/partclassifier.git
cd partclassifier
pip install -r requirements.txt
cp .env.example .env              # set LLM_PROVIDER and LLM_API_KEY
streamlit run app.py

Option C: Command Line (API key)

python main.py
python main.py --no-cache          # bypass LLM cache
python main.py --clear-cache       # delete cache before run

Execution Modes

API Key CLI (main.py)

  • Sequential processing with asyncio
  • Configurable LLM provider via .env (7 providers)
  • Resume capability via progress.json checkpoint
  • Periodic Excel writes every 10 parts
  • CLI flags: --input, --output, --no-cache, --clear-cache

Claude Code CLI (main_cc.py)

  • Zero API key requirement (uses claude CLI on PATH)
  • Batch classification: 100 parts per CLI call
  • Parallel workers (default 4, configurable with --workers)
  • Resume via progress_cc.json
  • Ctrl+C safe (saves progress + partial output)

Streamlit Web UI (app.py)

  • Interactive provider/model configuration in sidebar
  • File picker with data preview
  • Live progress bar and download buttons

Output Format

Each output Excel contains:

  • All original input columns
  • Part Class -- the classified category
  • Source URL -- the web page or API the attributes came from
  • Extracted attribute columns specific to that class (canonical names, ordered per schema)
output/
  Flat Washer.xlsx
  Split Lock Washer.xlsx
  Deep Groove Ball Bearing.xlsx
  Tube Fitting.xlsx

Reliability Features

  • Multi-tier search: APIs → stealth browser → cache → DuckDuckGo → fallback
  • Deterministic classification: 90+ class aliases with specificity resolution
  • Regex + LLM agreement tracking: measures extraction confidence
  • Canonical normalization: 500+ alias mappings for consistent columns
  • Unit conversion: inches ↔ mm per part's unit
  • Retry logic: exponential backoff for timeouts (429, 503)
  • CloakBrowser: 33 C++ patches, bypasses Cloudflare/reCAPTCHA
  • Metrics tracking: quality, cache effectiveness, regex/LLM agreement
