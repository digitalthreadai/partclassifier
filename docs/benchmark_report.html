Key Insights

Groq (llama-3.3-70b)

  • Fastest by 10x (~5s/part vs ~50s/part for Claude Code CLI)
  • Best single-part score: HGH20CA got 16 attrs (highest across all models)
  • 50% of parts unclassified due to network timeouts during parallel run
  • When scraping succeeds, classification and extraction quality is excellent
  • Free tier but requires external API key (not enterprise-compatible)

Claude Sonnet

  • Highest classification rate: 75% (33/44 parts classified from web content)
  • Some misclassifications: PLF010 (filter) as "O-Ring", HSR20R (linear guide) as "Ball Screw"
  • 70% of parts got attributes (reliable extraction)
  • Enterprise-friendly: no external API keys, runs via Claude Code CLI
  • Best budget option for enterprise deployments

Claude Opus

  • Highest attribute extraction: 80% of parts got attributes
  • Richest data: 4.1 avg attrs/part (32% more than Sonnet's 3.1)
  • Best at extracting from limited content - finds more attrs from same source
  • Most consistent: rarely returned 0 attrs when content was available
  • Slightly faster than Sonnet (48s vs 52s per part)

Recommendations

  • Enterprise (no API keys): Opus for best quality, Sonnet for budget
  • Standalone/dev: Groq is free and 10x faster (run solo, not parallel)
  • Web classification works: 75% classified without Part Name column
  • Bottleneck is scraping: ~80% of time spent on web scraping, not LLM
  • Parallel runs hurt: Run benchmarks sequentially to avoid network contention

Test Configuration

Environment

  • Machine: Windows 11, Python 3.11+
  • Input: 44 parts, 2 per manufacturer, no Part Name (Mfg PN + Mfg Name only)
  • Manufacturers: McMaster-Carr, SMC, THK, NTN, Swagelok, Parker, Keyence, Omron, Festo, Hiwin, TE Connectivity, Misumi, CKD, Edwards, VAT Group, Entegris, Volvo, IKO, Pall, MKS Instruments, Specialty Bolt, NSK
  • Web scraping: curl_cffi (TLS fingerprint) + CloakBrowser stealth + DuckDuckGo fallback
  • Classification: Deterministic keyword extraction from web content (class_extractor.py) + LLM fallback
  • All 3 benchmarks ran in parallel, causing network contention (Groq most affected)
