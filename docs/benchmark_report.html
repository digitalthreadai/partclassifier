Pipeline Improvements (Baseline → V8)

1 Hybrid Extraction Prompt

Schema hints combined with maximize-extraction instructions. The LLM gets class-specific attribute names as guidance while also being encouraged to extract any additional specs found.

2 Excel-Based Schema

Expanded to 81 part classes with 169 attribute aliases. Schema loaded from structured Excel instead of hardcoded Python dicts, making it maintainable by non-developers.

3 Deterministic Web Classification

Rule-based classifier identifies part class from web page content without LLM calls. Saves tokens and ensures consistent classification across runs.

4 Regex Pre-Extraction

Regex patterns extract obvious specs (dimensions, materials, tolerances) before LLM processing. LLM then validates and enriches rather than starting from scratch.

5 URL Cache with Eviction

Bad URLs (403s, empty pages, bot blocks) are automatically evicted from the cache. Second run (V8c) achieves 93% coverage by reusing good URLs and retrying failed ones.

6 Content Cleaner

HTML table extraction and smart truncation preserve structured spec tables while stripping navigation, ads, and boilerplate. Better signal-to-noise ratio for the LLM.

7 21 Bug Fixes

Production hardening: encoding fixes, timeout handling, retry logic, empty-response guards, schema key normalization, and edge-case coverage for unusual part numbers.
