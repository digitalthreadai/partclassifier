Key Findings

Recommendations

1 Use Groq for Maximum Extraction

Groq (main.py) with DuckDuckGo + CloakBrowser delivers 3x more attributes than Claude-based pipelines. llama-3.3-70b excels at structured extraction from raw HTML.

2 CloakBrowser Essential for McMaster-Carr

McMaster-Carr aggressively blocks bots. CloakBrowser stealth scraping is the only path that consistently returns spec pages for 92xxx / 98xxx / 99xxx part numbers.

3 Adaptive Search Eliminates Domain Lists

Adaptive search dynamically constructs better queries when initial scraping fails. Achieves 100% coverage on re-test without maintaining hardcoded trusted domain lists.

4 For Enterprise (No API Keys): Opus > Sonnet

When Groq API is unavailable, Opus outperforms Sonnet on this part set: 42% vs 38% coverage, 2.2 vs 1.7 avg attributes. Worth the higher token cost.

5 Add Electronic Component Classes

Resistors, potentiometers, and varistors need dedicated class schemas in ClassificationSchema.xlsx. Current schema gaps limit attribute extraction for these part types.
